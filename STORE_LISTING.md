### Detailed Description

```text
Patina Web Sync automatically adds the current non-private website context to time records in the free Patina Windows desktop app.

After you enable Web Sync in Patina and enter its local port and token in the extension, Patina Web Sync automatically synchronizes the active regular HTTP or HTTPS tab when the browser starts, the active tab changes, the page changes, the window regains focus, or a lightweight refresh runs. You can also select “Sync current page” at any time.

For each synced tab, the extension sends the complete page URL (including path, query, and fragment), page title, website icon information, and an incognito=false marker to Patina on the same computer. Patina stores the URL locally so it can be included in data exports. A URL query may contain search terms or other information entered into a website. Chromium-family builds also send a generated browser client identifier, browser kind, and extension version. Firefox sends those optional technical fields only if you grant the optional technical and interaction data permission. The local bearer token is used only to authenticate the request to your local Patina app.

All sync requests are limited to 127.0.0.1 or localhost. Synced webpage data is not sent to the developer, a cloud service, or a third-party server.

Incognito, private, and InPrivate tabs are skipped before the extension resolves the website icon or creates a sync payload. Browser internal pages and other non-HTTP pages are also skipped.

Patina Web Sync does not read page body or DOM content, form values, passwords, cookies, screenshots, clipboard contents, download history, or the browser history database.

Key features:
• Automatically sync the current regular website to local Patina.
• Keep website activity records on your own computer.
• Pair with Patina through a local port and token, without an account.
• Show connection and sync status, with a manual sync action when needed.
• Preserve the complete page URL for local Patina data exports.
• Skip private browsing and browser internal pages before sending.

Patina Web Sync requires the Patina Windows desktop app with Web Sync enabled.
```

## Data Processing Matrix

| Data | Source | Purpose | Destination and retention | Store classification |
| --- | --- | --- | --- | --- |
| Local port | User input | Construct the local Patina endpoint | Kept in extension local storage; not a payload field | Local configuration / technical data |
| Bearer token | User input | Authenticate the local request | Kept in extension local storage and sent only in the `Authorization` header to local Patina | Authentication information |
| Complete page URL | Active-tab URL | Identify the active page and support the local `url` export field | Sent to and stored by local Patina; domain fields are derived for classification | Browsing activity / web history; search terms may appear in queries |
| Page title | Active-tab metadata | Give the local time record useful website context | Sent to and retained by local Patina according to user controls | Website content |
| Website icon data or URL | Chromium local favicon cache or Firefox tab metadata | Display the website icon in Patina | Sent to and retained by local Patina | Website content |
| `incognito: false` | Active-tab metadata | Preserve protocol compatibility and confirm the tab is non-private | Sent to local Patina; private tabs return before payload construction | Browsing activity safeguard |
| Browser client identifier | Extension-generated UUID | Distinguish local browser clients | Chromium: sent to local Patina. Firefox: sent only after optional consent | Technical and interaction data |
| Browser kind and extension version | User agent and manifest | Local compatibility and diagnostics | Chromium: sent to local Patina. Firefox: sent only after optional consent | Technical and interaction data |
| Language preference | User selection | Localize extension UI | Kept only in extension local storage | Local setting; not transmitted |
| Recent sync status | Local bridge result | Show connection state | Kept only in extension local storage | Local interaction state; not analytics |

The extension never sends tab ID, window ID, capture timestamp, or event reason. It preserves the complete URL, so a query string can include search terms; this is disclosed even though the extension does not separately inspect or classify the query.

## Permission Justifications

### Chromium and Microsoft Edge

- `tabs`: identify the active regular tab, obtain its complete URL, title, icon reference, and private-browsing state, and react when the active tab changes. The extension does not read page body content.
- `favicon`: read the browser's local favicon cache so local Patina can display the website icon without fetching it from a remote server.
- `storage`: keep the local port, token, generated client identifier, language preference, and recent connection status on the user's computer.
- `alarms`: perform a lightweight 30-second refresh so Patina does not retain stale active-site context when browser events are missed.
- `http://127.0.0.1/*` and `http://localhost/*`: send authenticated Web Sync requests only to Patina running on the same computer.

### Firefox

- `tabs`, `storage`, `alarms`, and the two local host permissions have the same purposes as above.
- Firefox does not request Chromium's `favicon` permission; it uses icon metadata already exposed for the active tab.
- Required built-in data permissions: `authenticationInfo`, `browsingActivity`, `searchTerms`, and `websiteContent`.
- Optional built-in data permission: `technicalAndInteraction`. Core website sync continues if the user declines it.
- Minimum Firefox version: 142.0. Firefox Desktop supports built-in consent from 140, but the conservative 142 floor also satisfies AMO's Android compatibility validation without declaring an Android target.
