# Store Reviewer Test Instructions

Verified for candidate `0.2.0` on July 10, 2026.

## Product and Test Boundary

Patina Web Sync is a browser companion for the free Patina Windows desktop app. It automatically sends the current non-private active website context to Patina on the same computer so Patina can add website context to local time records.

There is no account, test login, remote server, cloud sync, payment, or subscription. Reviewers generate their own local port and token in Patina; no credential needs to be requested from the developer.

The extension connects only to `127.0.0.1` or `localhost`. Firefox requires version 142.0 or later.

## Requirements

- Windows and the Patina desktop app.
- The submitted Patina Web Sync browser package.
- A normal browser window and the public test page `https://example.com/`.

## Initial Setup

1. Install and open Patina.
2. Open Patina Settings and enable Web Sync.
3. Copy the local Web Sync port and token shown by Patina.
4. Install the submitted extension package.
5. Open the extension options page.
6. Enter the Patina port and token and save them.
7. Confirm the options page reports that the settings are saved. Do not send the token to the developer.

Expected result:

- The port and token remain in browser extension local storage.
- The token is used only as the bearer credential for requests to local Patina.
- No account or remote service is contacted.

## Automatic Sync

1. In a normal browser window, open `https://example.com/?review=test#full-url`.
2. Switch to another normal tab and back, or move focus away from and back to the browser window.
3. Wait briefly for the automatic sync event.
4. Open the extension popup.
5. Check the Patina activity view.

Expected result:

- The popup reports a synced state.
- Patina shows local website context for `example.com`.
- The extension sends the complete URL `https://example.com/?review=test#full-url`, page title, website icon information, and `incognito: false`.
- A Patina data export that includes the `url` / “URL 地址” field contains that complete URL, including its query and fragment.
- The extension does not send tab ID, window ID, capture timestamp, or event reason.

Automatic sync can also run when the browser or extension starts, the active page changes, a local setting changes, or the 30-second lightweight refresh detects stale context.

## Manual Sync

1. Keep `https://example.com/` active in a normal window.
2. Open the popup.
3. Select **Sync current page**.

Expected result:

- The same local-only payload and privacy checks are used.
- The popup reports success only after local Patina returns an HTTP success response whose JSON body explicitly contains `{ "ok": true }`.

## Network Verification

The only Web Sync request is:

```http
POST http://127.0.0.1:<port>/web-activity
Authorization: Bearer <token generated by local Patina>
Content-Type: application/json
```

`localhost` may be used instead of `127.0.0.1`. No remote destination is permitted by the manifests or content security policy.

Negative success checks:

- A plain `2xx` response without valid JSON must not show as synced.
- A JSON response without `ok: true` must not show as synced.
- `{ "ok": false }`, a failed HTTP status, or an unreachable port must not show as synced.

## Privacy and Failure Checks

### Private Window

1. If the browser requires it, enable the extension for incognito/private windows.
2. Open `https://example.com/` in an incognito, private, or InPrivate window.
3. Open the popup.

Expected result:

- The popup says the private window is not synced.
- It does not display the private page's title or domain.
- The extension returns before favicon resolution or payload construction.
- Patina receives no Web Sync request and stores no record for that private page.

### Browser Internal Page

1. Open `chrome://extensions`, `edge://extensions`, `about:addons`, or another browser-internal page.
2. Open the popup.

Expected result: the page is unsupported or not synced, and no Web Sync request is sent.

### Missing Token

Clear the token in the options page.

Expected result: the extension reports that setup is needed and does not send a Web Sync request.

### Wrong Port

Enter a local port where Patina is not listening.

Expected result: the extension reports an error or not-synced state and never reports success.

### Web Sync Disabled in Patina

Disable Web Sync in Patina while the extension remains configured.

Expected result: the extension reports that Patina Web Sync is disabled and does not treat the request as successful.

## Firefox 142+ Data Consent

Firefox declares these required built-in data categories:

- `authenticationInfo`: the local Patina bearer token.
- `browsingActivity`: the active page's complete URL.
- `searchTerms`: URL query strings can contain search terms; the extension preserves the full query for local URL export.
- `websiteContent`: the page title and website icon information.

Firefox declares `technicalAndInteraction` as optional. To test both paths:

1. Install the extension on Firefox 142 or later and review the built-in data consent information.
2. Decline or remove optional technical and interaction data access in **Permissions and Data**.
3. Repeat Automatic Sync.

Expected result: core website sync still succeeds, but the Firefox payload omits the browser client identifier, browser kind, and extension version.

4. Grant optional technical and interaction data access.
5. Repeat Automatic Sync.

Expected result: core sync still succeeds and those three local diagnostic fields may be sent to Patina.

The extension does not display a duplicate custom consent page.

## Browser Differences

- Chrome and Edge use the `favicon` permission to read the browser's local favicon cache; this avoids fetching a website icon from a remote server.
- Firefox does not request Chromium's `favicon` permission and uses active-tab icon metadata supplied by Firefox.
- Edge uses the validated Chromium-family package. Store screenshots, if supplied later, must be captured in Edge rather than presenting Chrome browser chrome as Edge.

## Data Not Read

The extension has no content scripts and does not read page body or DOM content, form values, passwords, cookies, screenshots, clipboard contents, download history, or the browser history database.
